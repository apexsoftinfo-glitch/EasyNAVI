# ZADANIE: 01_design_plan

## CEL
Zaplanuj **3 radykalnie różne style wizualne** dla ekranu startowego. Nie pisz jeszcze kodu ani nie zmieniaj żadnych plików na tym etapie.

W pliku `lib/features/home/ui/temporary_widgets/variants/home_variant_a.dart` masz zaakceptowany wireframe — interakcje, gesty i układ są już ustalone. Teraz nadajesz mu charakter wizualny.

W pliku `docs/skills/flutter-mobile-design-skill.md` masz opisane techniki, jak we Flutterze tworzyć ładnie wyglądające i ciekawie zachowujące się ekrany.

Wciąż pracujemy tylko na warstwie UI/UX. Nie łącz żadnych Cubitów, żadnej bazy danych, żadnej logiki, żadnych testów. Nie musisz przejmować się na tym etapie lokalizacją.

## PACZKI
Jeśli będziesz chciał dodać dodatkowe paczki (np. `google_fonts`, `flutter_animate` itp.), możesz to zrobić —  tylko zawsze instaluj aktualnie najnowsze wersje.

## KROKI DO WYKONANIA:
1. Przeczytaj `docs/IDEA.md` i `docs/skills/flutter-mobile-design-skill.md`.
2. Przeczytaj wireframe w `home_variant_a.dart`.
3. Zaplanuj **3 radykalnie różne style wizualne** (warianty D, E, F). Każdy musi być w **Light Mode** — Dark Mode będzie implementowany osobno w przyszłości jako feature Pro. Każdy musi być totalnie inny, ale wciąż pasujący stylistyką do tematu aplikacji. Myśl w kontekście kolorów, spacingu, głębi, doboru czcionek i ogólnego vibe'u. Każdy wariant powinien mieć własne, niezależne `ThemeData`.
4. Przedstaw propozycje użytkownikowi. Nie rekomenduj żadnego — user sam wybierze jeden po implementacji wszystkiego.
5. ZATRZYMAJ SIĘ i czekaj na `next`.

## FINISH
