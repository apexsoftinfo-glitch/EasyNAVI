# Intro

Twoim zadaniem jako agenta AI jest zrealizowanie poniższych poleceń.

CEL: Zawęzić grupę docelową aplikacji do konkretnej niszy.

# Kontekst

Istniejąca konkurencja = potwierdzony rynek, nie problem. Nie rób aplikacji dla wszystkich — zawęź grupę odbiorców. Przykład: właściciele zwierząt → właściciele psów → właściciele dalmatyńczyków.

# Rola

To Ty proponujesz zawężenia i uzasadniasz je. Użytkownik wybiera lub koryguje.

# Task

1. Na podstawie ustalonego pomysłu na aplikację zaproponuj **4 warianty zawężenia grupy docelowej**.

2. Każdy wariant przedstaw w formacie:
   - **Grupa docelowa**: dla kogo dokładnie jest ta aplikacja
   - **Dlaczego ta nisza**: 1-2 zdania — dlaczego ta grupa ma sens, jaka jest przewaga nad ogólną wersją
   - **Przykład dopasowania**: jak aplikacja byłaby lepiej dopasowana do tej grupy (design, funkcje, język)

3. Wskaż **1 wariant, który rekomendujesz najbardziej** i krótko uzasadnij wybór.

4. Daj też opcję: **"Zostawiam globalnie"** — jeśli użytkownik uważa, że zawężanie nie ma sensu dla tego pomysłu.

5. Poproś użytkownika, żeby wybrał wariant, zmodyfikował lub zostawił globalnie.

## Zasady

* Nie zadawaj więcej niż jednego pytania naraz.
* To Ty proponujesz zawężenia, nie pytaj użytkownika "jaka jest Twoja grupa docelowa?"
* Zawężenia mają być realistyczne i powiązane z pomysłem z poprzedniego kroku.
* Pamiętaj, że użytkownik też ma być użytkownikiem tej aplikacji — zawężenie musi obejmować jego samego.
* Nie przesadzaj z zawężeniem — "właściciele dalmatyńczyków w Krakowie powyżej 40 lat" to za dużo.

# Finish

Gdy użytkownik wybierze lub zatwierdzi grupę docelową (lub świadomie zdecyduje się na wersję globalną), potwierdź wybór i zapamiętaj go na potrzeby kolejnych kroków.

Gdy odpowie "next", przejdź do wykonania polecenia zawartego w `docs/commands/01_start/05_start-long-tail-keyword.md`.
