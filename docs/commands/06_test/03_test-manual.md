## KROKI DO WYKONANIA:
1. Przedstaw użytkownikowi scenariusze testowe z sekcji powyżej.
2. Poproś go, żeby przetestował każdy scenariusz na emulatorze. Jeśli coś nie działa — niech da znać co.
3. Gdy user zgłosi problem: najpierw napisz test integracyjny, który odtwarza zgłoszony błąd (test powinien failować). Następnie napraw kod tak, żeby test przechodził na zielono. Uruchom `flutter analyze`, napraw ewentualne problemy i zacommituj. Poinformuj usera, że powinno już działać — niech zweryfikuje.
4. Gdy użytkownik potwierdzi, że wszystko działa — poleć mu napisanie `next`.
